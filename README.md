# Slime Escape
A 2D microgame developed during a university game jam, where the player controls a slime attempting to escape a laboratory while avoiding obstacles and hazards.

The project focuses on fast-paced gameplay, simple but responsive controls, and delivering a complete experience within a short development timeframe.



## Key Points
- Developed player movement with soft body physics inspired player.
- Created simple but readable level layouts suited for quick play sessions
- Focused on rapid prototyping and iteration during a constrained game jam
- Delivered a polished, playable experience within a limited timeframe
## Tech
- Engine: Unity
- Language: C#
- Key systems: Player movement, obstacle interaction, simplistic design
## What I Learned / Would Improve
- Strengthened my ability to scope and deliver within tight deadlines
- Improved rapid prototyping and iteration workflows
- With more time, I would improve replay variety and refine difficulty balancing
- I would also enhance visual feedback and juice to improve game feel
